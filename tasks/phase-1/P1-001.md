# P1-001: Enforce business hours in availability engine

## Metadata
| Field | Value |
|-------|-------|
| Phase | 1 |
| Status | pending |
| Priority | P0 |
| Estimate | 30m |
| Dependencies | — |

## Objective

The availability engine (`src/lib/availability.ts`) only checks StaffSchedule for availability. It completely ignores BusinessHours. If a business is "closed on Sunday" but a staff member has a Sunday schedule, the system allows bookings on Sunday. Fix this by checking BusinessHours before generating slots.

## Acceptance Criteria

- [ ] `getAvailability()` checks BusinessHours for the given location and day
- [ ] If the business is closed that day (`isClosed: true` or no hours entry), return zero slots
- [ ] If the business has hours, staff availability is constrained to the intersection of staff schedule AND business hours
- [ ] `pnpm build` passes

## Implementation Notes

In `getAvailability()`, after fetching the staff schedule, also fetch BusinessHours:
```typescript
const businessHours = await prisma.businessHours.findUnique({
  where: { locationId_dayOfWeek: { locationId: params.locationId, dayOfWeek } }
})

// If closed, return no slots
if (!businessHours || businessHours.isClosed || !businessHours.openTime || !businessHours.closeTime) {
  return { slots: [], staffId: params.staffId, serviceId: params.serviceId, date: dateStr, serviceDuration }
}

// Constrain slot generation to the intersection of staff schedule hours and business hours
// e.g., if business opens 9am-7pm and staff works 8am-5pm, effective range is 9am-5pm
```

The `locationId` is already available as a parameter to `getAvailability()`.
