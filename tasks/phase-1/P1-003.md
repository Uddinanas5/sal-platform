# P1-003: Add rate limiting to public booking cancellation

## Metadata
| Field | Value |
|-------|-------|
| Phase | 1 |
| Status | pending |
| Priority | P0 |
| Estimate | 15m |
| Dependencies | — |

## Objective

`cancelPublicBooking()` in `src/lib/actions/public-booking.ts` has no rate limiting. Add rate limiting to prevent abuse (bulk cancellation attacks).

## Acceptance Criteria

- [ ] Cancellation limited to 3 attempts per booking reference per 10 minutes
- [ ] Uses the existing `rateLimit()` utility from `src/lib/rate-limit.ts`
- [ ] Returns appropriate error when rate limited
- [ ] `pnpm build` passes

## Implementation Notes

Add at the top of `cancelPublicBooking()`:
```typescript
const rateLimitResult = rateLimit(`cancel:${bookingReference}`, 3, 10 * 60 * 1000)
if (!rateLimitResult.success) {
  return { success: false, error: "Too many cancellation attempts. Please try again later." }
}
```
