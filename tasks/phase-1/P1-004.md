# P1-004: Guard email sending when RESEND_API_KEY is missing

## Metadata

| Status | pending |

## Objective

`src/lib/email.ts` creates a Resend client even when `RESEND_API_KEY` is undefined, causing confusing runtime errors when emails are sent. Add a guard that gracefully skips email sending with a console warning.

## Acceptance Criteria

- [ ] If `RESEND_API_KEY` is not set, `sendEmail()` returns `{ success: false, error: "Email not configured" }` without throwing
- [ ] A console warning is logged once on startup when the key is missing
- [ ] App functions normally without Resend configured (bookings still work, just no emails)
- [ ] `pnpm build` passes

## Implementation Notes

```typescript
const apiKey = process.env.RESEND_API_KEY
if (!apiKey) {
  console.warn("RESEND_API_KEY not set — emails will not be sent")
}
export const resend = apiKey ? new Resend(apiKey) : null

export async function sendEmail(...) {
  if (!resend) {
    console.warn("Email skipped (Resend not configured):", to, subject)
    return { success: false, error: "Email service not configured" }
  }
  // ... existing logic
}
```
