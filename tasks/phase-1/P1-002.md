# P1-002: Make password reset tokens single-use

## Metadata

## Objective

Password reset tokens (JWT with 1-hour expiry) can currently be reused unlimited times within the validity window. Make them single-use by tracking consumption in the database.

## Acceptance Criteria

- [ ] After a successful password reset, the same token cannot be used again
- [ ] A new `passwordResetToken` and `passwordResetExpiry` field on the User model tracks the active token
- [ ] `resetPassword()` verifies the token matches the stored one before proceeding
- [ ] After reset, the stored token is cleared
- [ ] `pnpm build` passes

## Implementation Notes

Option A (simplest, no migration): Store a `passwordResetNonce` in the JWT payload and in `User.metadata` JSON field. On reset, verify the nonce matches, then clear it.

Option B (cleaner): Add `passwordResetToken` and `passwordResetExpiry` fields to the User model. On `requestPasswordReset()`, store a random nonce. On `resetPassword()`, verify the nonce. On success, null both fields.

Recommend Option A to avoid schema migration for production deployment.
