## Objective

Display times in the business's local timezone throughout the public booking flow, and ensure the correct UTC time is sent to the server when creating a booking.

## Acceptance Criteria

- [ ] Business timezone is fetched and passed to the booking client
- [ ] All time slots in DateTimeStep display in the business's local timezone
- [ ] The confirmation step shows the appointment time in the business's timezone
- [ ] The correct UTC time is sent to `createPublicBooking()` (no double-conversion)
- [ ] A small timezone indicator (e.g., "Times shown in EST") is displayed near the time slots
- [ ] `pnpm build` passes

## Context

- **Current state**: All times are stored and displayed in UTC. A salon in New York shows raw UTC times to customers, which could be hours off.
- **Business timezone**: The onboarding flow collects timezone (`updateBusinessDetails` action saves it). The `Business` model has a `timezone` field (or it's in `Location`). Need to verify where it's stored.
- **Availability engine**: `src/lib/availability.ts` works with `Date` objects. The API returns ISO strings. The client needs to format these in the business timezone.
- **Key constraint**: The server stores everything in UTC. The client only needs to _display_ in local time. No server-side timezone logic changes needed if the availability API already returns correct UTC slots.

## Implementation Notes

### Step 1: Verify timezone storage

Check where business timezone is stored — likely `Business.timezone` or `Location.timezone`. The onboarding action `updateBusinessDetails` saves a timezone field.

### Step 2: Pass timezone to client

In `src/app/book/[businessSlug]/page.tsx`, include the business timezone in props:
```typescript
const timezone = business.timezone || location?.timezone || 'America/New_York'
// Pass to: <BookingPageClient timezone={timezone} ... />
```

### Step 3: Format display times

Use `Intl.DateTimeFormat` or a lightweight helper to format UTC ISO strings in the business timezone:
```typescript
function formatTimeInTimezone(isoString: string, timezone: string): string {
  return new Date(isoString).toLocaleTimeString('en-US', {
    hour: 'numeric',
    minute: '2-digit',
    timeZone: timezone,
  })
}
```

Apply this to:
- Time slot buttons in DateTimeStep
- Selected time display in ConfirmationStep
- Success screen time display

### Step 4: Add timezone indicator

Below the time slots grid, show:
```
All times shown in Eastern Time (ET)
```
Use `Intl.DateTimeFormat` to get the timezone abbreviation.

### Step 5: Calendar date handling

Ensure the date picker selects dates in the business timezone context. When the user picks "March 20", the API call should use "2026-03-20" in the business's local timezone, not the browser's timezone.
