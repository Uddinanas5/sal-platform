# P0-011: Surface waitlist option in public booking when no slots available

## Metadata
| Field | Value |
|-------|-------|
| Phase | 0 |
| Status | pending |
| Priority | P2 |
| Estimate | 30m |
| Dependencies | P0-002 |
| Blocks | — |

## Objective

When no time slots are available for a selected date (or date range), offer the customer the option to join a waitlist so they can be notified when a slot opens up.

## Acceptance Criteria

- [ ] When `/api/availability` returns zero slots for a date, show a "Join Waitlist" option instead of just "No available time slots"
- [ ] Waitlist form collects: preferred date, preferred time range (morning/afternoon/evening), name, email, phone
- [ ] Submission calls the existing `addToWaitlist()` server action
- [ ] Success message confirms they've been added to the waitlist
- [ ] `pnpm build` passes

## Context

- **Existing infrastructure**: `WaitlistEntry` model exists in the Prisma schema with fields: `businessId`, `clientId`, `serviceId`, `staffId`, `preferredDate`, `preferredTimeStart`, `preferredTimeEnd`, `status`, `notes`.
- **Existing action**: `src/lib/actions/waitlist.ts` has `addToWaitlist()` which creates a waitlist entry.
- **Existing query**: `src/lib/queries/waitlist.ts` has `getWaitlistEntries()`.
- **Current gap**: The public booking flow shows "No available time slots for this day" with no alternative.

## Implementation Notes

### Step 1: Detect zero-slot state

In the DateTimeStep (after P0-002 integration), when `availableSlots.length === 0 && !slotsLoading`:

Show a friendly message with CTA:
```
No available slots for this date.
Want to be notified when a spot opens up?
[Join Waitlist]
```

### Step 2: Waitlist form

When "Join Waitlist" is clicked, show an inline form:
- Preferred time: Morning (9am-12pm) / Afternoon (12pm-5pm) / Evening (5pm-9pm) / Any
- Name (pre-fill from details step if already entered)
- Email (required)
- Phone (optional)
- Notes (optional)

### Step 3: Submit to waitlist action

The existing `addToWaitlist()` may require auth (business context). If so, create a `addToPublicWaitlist()` action that:
- Takes businessId, serviceId, staffId (optional), preferredDate, preferredTimeStart/End, client details
- Finds or creates client record (same as `createPublicBooking`)
- Creates `WaitlistEntry` with status `waiting`
- Returns success

### Step 4: Success state

After submission, show confirmation:
```
You've been added to the waitlist!
We'll email you at {email} when a slot opens up for {date}.
```

Offer to try a different date or go back.
