## Objective

Replace the naive local time slot generation in the public booking flow with real calls to the `/api/availability` endpoint, so customers only see genuinely available time slots.

## Acceptance Criteria

- [ ] DateTimeStep fetches real availability from `/api/availability` when a date is selected
- [ ] Time slots reflect actual staff schedules, existing appointments, breaks, and time-off
- [ ] Loading state shown while availability is being fetched
- [ ] Empty state ("No available slots") shown when API returns zero slots
- [ ] Local `generateTimeSlots()` and `getTimeSlotsForDay()` helper functions are removed
- [ ] Selected staff ID is passed to the API (or omitted for "any available" — handled in P0-003)
- [ ] `pnpm build` passes

## Context

- **Problem file**: `src/app/book/[businessSlug]/client.tsx`
- **Current behavior**: `generateTimeSlots(openHour, openMinute, closeHour, closeMinute)` creates 30-min intervals from business hours only. Ignores existing appointments, staff schedules, breaks, and time-off. Customers can select already-booked slots and only get an error at final submission.
- **Availability API**: `GET /api/availability?serviceId=X&date=YYYY-MM-DD&locationId=Y&staffId=Z` already exists and returns real availability. Response shape:
  ```json
  {
    "allSlots": [{ "startTime": "ISO", "displayTime": "9:00 AM", "availableStaff": ["id1"], "staffCount": 1 }],
    "totalAvailableSlots": N
  }
  ```
- **Availability engine**: `src/lib/availability.ts` handles service duration, buffer times, staff breaks, time-off, existing appointments, and 30-min lead time for same-day bookings.

## Implementation Notes

### Step 1: Add state for async slot fetching

Add to BookingPageClient:
```typescript
const [availableSlots, setAvailableSlots] = useState<Array<{startTime: string, displayTime: string}>>([])
const [slotsLoading, setSlotsLoading] = useState(false)
```

### Step 2: Fetch availability on date selection

When user selects a date in DateTimeStep, call the API:
```typescript
async function fetchAvailability(date: Date) {
  setSlotsLoading(true)
  setAvailableSlots([])
  setSelectedTime(null)

  const dateStr = `${date.getFullYear()}-${String(date.getMonth()+1).padStart(2,'0')}-${String(date.getDate()).padStart(2,'0')}`
  const params = new URLSearchParams({
    serviceId: selectedService.id,
    date: dateStr,
    locationId: locationId, // need to pass this from page.tsx
  })
  if (selectedStaff !== 'any') {
    params.set('staffId', selectedStaff)
  }

  const res = await fetch(`/api/availability?${params}`)
  const data = await res.json()
  setAvailableSlots(data.allSlots ?? [])
  setSlotsLoading(false)
}
```

### Step 3: Pass locationId from page.tsx

The server page already fetches the primary location but doesn't pass its ID. Add `locationId` to the props passed to BookingPageClient.

### Step 4: Update DateTimeStep rendering

Replace the current `useMemo`-based slot generation with the `availableSlots` state. Show a spinner while `slotsLoading`, and "No available time slots" when slots are empty and not loading.

### Step 5: Clean up dead code

Remove `generateTimeSlots()`, `getTimeSlotsForDay()`, `parseTimeFromISO()`, and related `useMemo` computations that are no longer needed.

### Step 6: Update time selection handler

When a slot is selected, store the ISO `startTime` from the API response (not a locally constructed time string). This ensures the time sent to `createPublicBooking()` matches exactly what the availability engine validated.
