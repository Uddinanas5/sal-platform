## Objective

Connect the existing BookingSettingsTab UI to the new `updateBookingSettings()` server action so admin changes are actually persisted and loaded on page refresh.

## Acceptance Criteria

- [ ] Settings are loaded from the database on component mount (via props from server page)
- [ ] All form fields (lead time, max advance, cancellation window, double booking, auto-confirm, deposit settings, required fields, custom questions) are initialized from saved values
- [ ] "Save" button calls `updateBookingSettings()` and shows success/error toast
- [ ] Settings persist across page reloads
- [ ] `pnpm build` passes

## Context

- **Component**: `src/components/settings/booking-settings-tab.tsx` (365 lines)
- **Current state**: All settings are local `useState` with hardcoded defaults. `handleSave()` just calls `toast.success("Booking settings saved!")`.
- **After P0-004**: `updateBookingSettings()` and `getBookingSettings()` will be available.
- **Settings page**: `src/app/(dashboard)/settings/page.tsx` is the server component that passes data to settings tabs. It will need to fetch booking settings and pass them as props.

## Implementation Notes

### Step 1: Pass settings from server page

In `src/app/(dashboard)/settings/page.tsx`, fetch booking settings:
```typescript
const bookingSettings = await getBookingSettings(businessId)
// Pass to client: <SettingsClient bookingSettings={bookingSettings} ... />
```

### Step 2: Accept settings as props

Update `BookingSettingsTab` to accept `initialSettings` prop and initialize state from it instead of hardcoded defaults.

### Step 3: Wire save button

Replace `handleSave()`:
```typescript
async function handleSave() {
  setIsSaving(true)
  const result = await updateBookingSettings({
    minLeadTime, maxAdvanceBooking, cancellationWindow,
    autoConfirm, allowDoubleBooking,
    requireDeposit, depositType,
    depositAmount: parseFloat(depositAmount),
    depositApplyOverAmount: parseFloat(applyOverAmount),
    requiredFields, customQuestions,
  })
  if (result.success) toast.success("Booking settings saved!")
  else toast.error(result.error)
  setIsSaving(false)
}
```

### Step 4: Add loading state to save button

Show spinner/disabled state while saving.
