## Objective

When a customer selects "Any Available" for staff, show combined availability across all qualifying staff and auto-assign the best available staff member at booking time.

## Acceptance Criteria

- [ ] "Any Available" calls `/api/availability` WITHOUT a `staffId` param to get combined slots
- [ ] Time slots show availability across all staff who provide the selected service
- [ ] Each slot internally tracks which staff members are available at that time
- [ ] At confirmation, the selected staff is resolved to a specific staff member from the available list
- [ ] If multiple staff are available for a slot, one is picked (first available or random)
- [ ] `pnpm build` passes

## Context

- **Current behavior**: When "Any Available" is selected, `handleConfirm()` just picks the first staff member from the full staff list who provides the service — with no availability check. Time slots are the same generic business-hours slots regardless.
- **API support**: `/api/availability` already handles the no-`staffId` case. It calls `getMultiStaffAvailability()` and returns combined slots with `availableStaff: [staffId1, staffId2, ...]` per time slot.

## Implementation Notes

### Step 1: Adjust fetchAvailability (from P0-002)

When `selectedStaff === 'any'`, omit the `staffId` param. The API response `allSlots` already includes `availableStaff` array per slot.

### Step 2: Store available staff per slot

Extend the slot state to track which staff can serve each slot:
```typescript
interface AvailableSlot {
  startTime: string    // ISO
  displayTime: string  // "9:00 AM"
  availableStaff: string[]  // staff IDs
  staffCount: number
}
```

### Step 3: Resolve staff at confirmation

In `handleConfirm()`, when `selectedStaff === 'any'`:
- Find the selected time slot's `availableStaff` array
- Pick the first staff ID (or distribute evenly for load balancing — simple first-pick is fine for now)
- Pass that resolved `staffId` to `createPublicBooking()`

### Step 4: Show staff count hint (optional UX improvement)

In the time slot button, optionally show how many staff are available (e.g., "9:00 AM · 3 available") to help customers pick well-staffed slots.
