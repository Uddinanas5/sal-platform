## Objective

Fix a multi-tenancy data leak in the dashboard booking page where `getServices()` and `getStaff()` are called without a `businessId`, causing all businesses' data to be returned.

## Acceptance Criteria

- [ ] `getServices()` and `getStaff()` are called with the current user's `businessId`
- [ ] `getBusinessContext()` is used to extract `businessId` from the session
- [ ] Only the current business's services and staff are shown on the booking page
- [ ] `pnpm build` passes

## Context

- **Bug location**: `src/app/(dashboard)/booking/page.tsx` (12 lines)
- **Root cause**: Page calls `getServices()` and `getStaff()` with no arguments. Both functions return ALL records across businesses when no `businessId` is provided.
- **Correct pattern**: The public booking page (`src/app/book/[businessSlug]/page.tsx`) correctly scopes by business. The dashboard page should use `getBusinessContext()` from `src/lib/auth-utils.ts`.

## Implementation Notes

```typescript
// Current (broken):
const [services, staff] = await Promise.all([getServices(), getStaff()])

// Fix:
import { getBusinessContext } from "@/lib/auth-utils"

const { businessId } = await getBusinessContext()
const [services, staff] = await Promise.all([
  getServices(businessId),
  getStaff(businessId)
])
```
