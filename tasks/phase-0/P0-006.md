# P0-006: Enforce booking settings in availability and public flow

## Objective

Make the saved booking settings (lead time, max advance booking, cancellation window) actually enforced in the availability engine and public booking UI.

## Acceptance Criteria

- [ ] Minimum lead time: slots within the lead time window are filtered out (e.g., if 4h lead time, no slots in the next 4 hours)
- [ ] Maximum advance booking: dates beyond the max window are disabled in the calendar (e.g., if 1 month, can't book 5 weeks out)
- [ ] The availability API reads business booking settings and applies lead time filtering
- [ ] The public booking client reads settings to disable calendar dates beyond max advance
- [ ] `pnpm build` passes

## Context

- **Availability engine**: `src/lib/availability.ts` currently has a hardcoded 30-minute lead time (line ~210). This should be replaced with the configurable `minLeadTime` from booking settings.
- **Calendar UI**: `src/app/book/[businessSlug]/client.tsx` DateTimeStep currently only disables past dates. It should also disable dates beyond the `maxAdvanceBooking` window.
- **Settings access**: The public booking page server component (`page.tsx`) should fetch `getPublicBookingSettings(businessId)` and pass relevant settings to the client.

## Implementation Notes

### Step 1: Pass booking settings to availability engine

Update `getAvailability()` in `src/lib/availability.ts` to accept an optional `minLeadTimeMinutes` param:
```typescript
// Convert setting string to minutes
const leadTimeMap = { "none": 0, "1h": 60, "2h": 120, "4h": 240, "12h": 720, "24h": 1440, "48h": 2880 }
```
Replace the hardcoded 30-minute lead time with this value.

### Step 2: Pass settings through availability API

In `src/app/api/availability/route.ts`, fetch the business booking settings and pass `minLeadTimeMinutes` to `getAvailability()`.

### Step 3: Pass max advance booking to client

In `src/app/book/[businessSlug]/page.tsx`, fetch booking settings and pass `maxAdvanceBooking` to the client component. In the calendar, disable dates beyond the max window:
```typescript
const maxAdvanceMap = { "1w": 7, "2w": 14, "1m": 30, "2m": 60, "3m": 90 }
const maxDate = new Date()
maxDate.setDate(maxDate.getDate() + maxAdvanceMap[bookingSettings.maxAdvanceBooking])
// Disable dates after maxDate in calendar
```

### Step 4: Handle cancellation window

The cancellation window is enforced in P0-007 (public booking management page) where cancel/reschedule actions check if the appointment is within the window.
