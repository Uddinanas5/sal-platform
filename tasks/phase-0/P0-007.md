## Objective

Create a public-facing page where customers can view, cancel, or reschedule their booking using their booking reference number — without requiring authentication.

## Acceptance Criteria

- [ ] Route: `/book/manage/[bookingReference]` displays appointment details
- [ ] Shows: service, staff, date/time, status, booking reference, business info
- [ ] "Cancel Appointment" button cancels the booking (with confirmation dialog)
- [ ] "Reschedule" button opens a date/time picker to select a new time
- [ ] Cancellation respects the cancellation window setting (if within window, show message instead)
- [ ] Cancelled appointments show a "cancelled" state instead of action buttons
- [ ] Past appointments show a "completed/past" state
- [ ] Invalid booking reference shows 404
- [ ] `pnpm build` passes

## Context

- **No existing route**: This is a new page. The `/book/` directory currently only has `[businessSlug]/`.
- **Existing server action**: `updateAppointmentStatus()` in `src/lib/actions/appointments.ts` handles status changes but requires auth. Need a public version.
- **Existing reschedule action**: `rescheduleAppointment()` exists but also requires auth.
- **Email integration**: P0-008 will add links to this page in the confirmation email.
- **Booking reference format**: `SAL-XXXXXXXX` (8 alphanumeric chars).

## Implementation Notes

### Step 1: Create route structure

```
src/app/book/manage/[bookingReference]/
├── page.tsx    # Server component - fetch appointment by reference
└── client.tsx  # Client component - UI with cancel/reschedule actions
```

### Step 2: Server page (page.tsx)

```typescript
// Fetch appointment by bookingReference (no auth required)
const appointment = await prisma.appointment.findUnique({
  where: { bookingReference: params.bookingReference },
  include: {
    business: { select: { name: true, slug: true, phone: true, email: true, settings: true } },
    client: { select: { firstName: true, lastName: true, email: true } },
    appointmentServices: {
      include: {
        service: { select: { name: true } },
        staff: { select: { user: { select: { name: true } } } }
      }
    }
  }
})
if (!appointment) notFound()
```

### Step 3: Client component (client.tsx)

Display a card with:
- Business name and logo
- Service name, staff name, date/time, duration
- Booking reference
- Status badge (confirmed, cancelled, completed, etc.)
- Action buttons based on status:
  - `confirmed`/`pending` → Cancel + Reschedule buttons
  - `cancelled` → "This appointment has been cancelled" message
  - `completed`/past → "This appointment has passed" message

### Step 4: Public cancel action

Create `cancelPublicBooking(bookingReference: string, clientEmail: string)` in `src/lib/actions/public-booking.ts`:
- Verify `bookingReference` exists
- Verify `clientEmail` matches the appointment's client email (lightweight auth)
- Check cancellation window from booking settings
- Update status to `cancelled`, set `cancelledAt`
- Send cancellation email
- Revalidate paths

### Step 5: Public reschedule action

Create `reschedulePublicBooking(bookingReference: string, clientEmail: string, newStartTime: string)` in `src/lib/actions/public-booking.ts`:
- Same verification as cancel
- Check new slot availability via `isSlotAvailable()`
- Update appointment `startTime`, `endTime`
- Send reschedule notification email
- Revalidate paths

### Step 6: Reschedule UI

When "Reschedule" is clicked, show a date/time picker (reuse availability fetching from P0-002 pattern). On selection, confirm and call reschedule action.

### Step 7: Add to middleware

Ensure `/book/manage/*` is in the public routes list in `src/middleware.ts`.
