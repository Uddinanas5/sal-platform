## Objective

Create the backend for persisting online booking configuration — lead time, max advance booking, cancellation window, deposit settings, required form fields, and custom questions — so admin settings are actually saved and enforceable.

## Acceptance Criteria

- [ ] Booking settings are stored per-business (use `Business.settings` JSON or a new model)
- [ ] Server action `updateBookingSettings()` validates and saves all booking config fields
- [ ] Server action `getBookingSettings()` (or query) retrieves current settings with sensible defaults
- [ ] Settings include: minLeadTime, maxAdvanceBooking, cancellationWindow, autoConfirm, allowDoubleBooking, requireDeposit, depositType, depositAmount, depositApplyOverAmount, requiredFields, customQuestions
- [ ] Zod validation on all input fields
- [ ] `pnpm build` passes

## Context

- **Current state**: `BookingSettingsTab` has full UI for all these settings but `handleSave()` just shows a toast — nothing persists.
- **Schema options**:
  - **Option A (recommended)**: Use the existing `Business.settings` JSON field. The schema already has `settings Json?` on the Business model. Onboarding already stores `{ isOnboarded: true }` here. Extend this with a `booking` key.
  - **Option B**: Create a new `BookingSettings` model. More type-safe but adds migration complexity.
- **Existing pattern**: `src/lib/actions/settings.ts` already has `updateBusinessSettings()` and `updateLocationSettings()`.

## Implementation Notes

### Step 1: Define the settings shape

```typescript
// src/lib/actions/booking-settings.ts (new file)

const bookingSettingsSchema = z.object({
  minLeadTime: z.enum(["none", "1h", "2h", "4h", "12h", "24h", "48h"]).default("none"),
  maxAdvanceBooking: z.enum(["1w", "2w", "1m", "2m", "3m"]).default("1m"),
  cancellationWindow: z.enum(["none", "1h", "2h", "4h", "12h", "24h", "48h"]).default("24h"),
  autoConfirm: z.boolean().default(true),
  allowDoubleBooking: z.boolean().default(false),
  requireDeposit: z.boolean().default(false),
  depositType: z.enum(["percentage", "fixed"]).default("percentage"),
  depositAmount: z.number().min(0).default(0),
  depositApplyOverAmount: z.number().min(0).default(0),
  requiredFields: z.object({
    phone: z.boolean().default(true),
    email: z.boolean().default(true),
    address: z.boolean().default(false),
    notes: z.boolean().default(false),
  }).default({}),
  customQuestions: z.array(z.string()).default([]),
})
```

### Step 2: Save action

```typescript
export async function updateBookingSettings(data: z.infer<typeof bookingSettingsSchema>) {
  const { businessId } = await getBusinessContext()
  const validated = bookingSettingsSchema.parse(data)

  // Read existing settings, merge booking key
  const business = await prisma.business.findUnique({ where: { id: businessId }, select: { settings: true } })
  const existingSettings = (business?.settings as Record<string, unknown>) ?? {}

  await prisma.business.update({
    where: { id: businessId },
    data: {
      settings: { ...existingSettings, booking: validated }
    }
  })

  revalidatePath("/settings")
  return { success: true, data: validated }
}
```

### Step 3: Read query

```typescript
export async function getBookingSettings(businessId: string) {
  const business = await prisma.business.findUnique({
    where: { id: businessId },
    select: { settings: true }
  })
  const settings = (business?.settings as Record<string, unknown>)?.booking
  return bookingSettingsSchema.parse(settings ?? {}) // returns defaults for missing fields
}
```
